================================================================================
Airtable Base 생성 가이드
================================================================================

본 문서는 Airtable에서 Base를 생성하는 방법을 단계별로 안내하는 문서입니다.

================================================================================
1. Base 생성 방법
================================================================================

1.1. Airtable 웹사이트 접속
- 브라우저에서 https://airtable.com 에 접속하세요
- 계정에 로그인하세요

1.2. Base 생성 시작
방법 1: 홈 화면에서 생성
1. 로그인 후 나타나는 홈 화면에서
2. 상단에 있는 "+ Add a base" 또는 "새 Base 추가" 버튼을 클릭하세요
3. 또는 왼쪽 사이드바에 있는 "+" 아이콘을 클릭하세요

방법 2: 템플릿에서 시작
1. "Templates" (템플릿) 메뉴를 클릭하세요
2. 원하는 템플릿을 선택하거나
3. "Blank base" (빈 Base)를 선택하세요

1.3. Base 이름 설정
1. "Name your base" 또는 "Base 이름을 입력하세요" 입력창이 나타납니다
2. 원하는 이름을 입력하세요 (예: "AX 역량 진단", "AX Competency")
3. "Create base" 또는 "Base 만들기" 버튼을 클릭하세요

1.4. Base 생성 완료
- Base가 생성되면 자동으로 첫 번째 테이블이 생성됩니다
- 기본 테이블 이름은 "Table 1" 또는 "테이블 1"입니다

================================================================================
2. Base 생성 후 확인
================================================================================

2.1. Base 목록 확인
- 왼쪽 사이드바에서 생성한 Base를 확인할 수 있어요
- Base 이름을 클릭하면 Base를 열 수 있어요

2.2. Base ID 확인 방법
Base ID는 나중에 config.js 파일에 입력해야 해요!

1. Base를 열어주세요
2. 우측 상단의 "Help" (도움말) 메뉴를 클릭하세요
   (또는 "?" 아이콘을 클릭)
3. "API documentation" (API 문서)를 선택하세요
4. 페이지 상단에 "Base ID"가 표시됩니다
   예: appAbc123Def456
5. 이 Base ID를 복사해서 보관하세요

================================================================================
3. 테이블 이름 변경 (선택사항)
================================================================================

기본 테이블 이름을 "사용자_평가결과"로 변경하는 방법:

1. Base를 열어주세요
2. 왼쪽 상단에 있는 테이블 이름("Table 1" 또는 "테이블 1")을 클릭하세요
3. 이름을 "사용자_평가결과"로 변경하세요
   (정확히 이 이름으로 입력해주세요 - 공백과 언더스코어 포함)
4. Enter 키를 누르거나 다른 곳을 클릭하면 저장됩니다

================================================================================
4. 다음 단계
================================================================================

Base 생성이 완료되면:

1. 테이블 이름을 "사용자_평가결과"로 변경하세요
2. Base ID를 확인하고 복사하세요
3. API 키를 발급받으세요 (다음 단계)
4. config.js 파일에 API 키와 Base ID를 입력하세요

6. 문제 해결
================================================================================

6.1. "+ Add a base" 버튼이 보이지 않아요
- 페이지를 새로고침해보세요 (F5)
- 브라우저 캐시를 삭제해보세요
- 다른 브라우저로 시도해보세요

6.2. Base를 만들 수 없어요
- 무료 플랜에서는 Base 개수 제한이 있을 수 있어요
- 기존 Base를 삭제하거나 업그레이드를 고려해보세요

6.3. Base ID를 찾을 수 없어요
- Base를 열어야 API documentation에 접근할 수 있어요
- Help 메뉴가 보이지 않으면 "?" 아이콘을 찾아보세요
- 또는 Base URL에서 직접 확인할 수 있어요:
  https://airtable.com/appXXXXXXXXXXXXXX
  여기서 "app" 뒤의 문자가 Base ID입니다

================================================================================
7. 팁
================================================================================

7.1. Base 이름
- 나중에 여러 Base를 만들 수 있으니 명확한 이름을 사용하세요
- 예: "AX 역량 진단", "AX Competency 2026" 등

7.2. Base ID 백업
- Base ID는 나중에 필요하니 안전한 곳에 저장해두세요
- config.js 파일에 입력하기 전에 복사해두세요

7.3. 테이블 이름
- 테이블 이름은 정확히 "사용자_평가결과"로 입력해야 해요
- 대소문자, 공백, 언더스코어를 정확히 맞춰주세요
